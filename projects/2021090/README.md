#### **Δυο λόγια για τα κριτήρια ταξινόμησης**

Τα χαρακτηριστικά αυτά επιλέχτηκαν με γνώμονα την προσαρμοστικότητα των κονσολών αυτών ως εργαλεία οπτικοποίησης για αποτελεσματική και ενημερωτική απεικονίσεων στα δι'αφορα είδη κοινού καθώς και πλατφόρμες έκθεσης αυτών.

* **Παλέτα χρωμάτων:** Το χρώμα παίζει σημαντικό ρόλο στην οπτική μετάδοση πληροφοριών. Μια πλούσια και διαμορφώσιμη παλέτα χρωμάτων παρέχει στους προγραμματιστές την ευελιξία να χρησιμοποιούν το χρώμα στρατηγικά στις απεικονίσεις τους. Σε διαφορετικά σημεία δεδομένων ή κατηγορίες μπορούν να εκχωρηθούν διαφορετικά χρώματα, ενισχύοντας τη σαφήνεια και το νόημα της οπτικοποίησης.

* **Πολαπλή προβολή:** Η ευελιξία στις επιλογές εμφάνισης επιτρέπει στους προγραμματιστές να προσαρμόζουν τις απεικονίσεις σε διάφορα περιβάλλοντα και συσκευές. Διασφαλίζει ότι οι οπτικοποιήσεις αποκρίνονται και είναι προσβάσιμες σε διαφορετικά μεγέθη οθόνης, αναλύσεις και πλατφόρμες. Αυτή η προσαρμοστικότητα είναι ζωτικής σημασίας για την οπτικοποίηση πληροφοριών, καθώς τα δεδομένα μπορούν να παρουσιαστούν σε ένα ευρύ φάσμα συσκευών και περιβαλλόντων.

* **Υποστήριξη πολλαπλών γλωσσών:** Η οπτικοποίηση πληροφοριών συχνά εξυπηρετεί ένα παγκόσμιο κοινό. Είναι σημαντικό για το περιβάλλον ανάπτυξης να υποστηρίζει πολλές γλώσσες, εξασφαλίζει έτσι την προσβασιμότητα και ευρύτερη προσέγγιση για οπτικοποιήσεις που βασίζονται σε δεδομένα.

## [Εβδομάδα Έκτη: cli data analysis]()

Ως άσκηση γραμμής εντολών επέλεξα να δημιουργήσω μερικά διαγράμματα. Ως data set χρησιμοποίησα ένα csv αρχείο που είχα από το μάθημα των δικτύων στο οποίο αποφάσισα να οπτικοποιήσω για άλλη μια φορά τα θεωρητικά και τα πειραματικά δεδομένα που υπήρχαν σε αυτο χρησιμοποιώντας αποκλειστικά εργαλεία γραμμής εντολών. Συγκεκριμένα χρησιμοποίησα το gnuplot και shell scripting. Δημιούργησα ένα μικρό shell script που να μου επιτρέπει να φτιάξω ένα αρχείο csv μονο με τις στήλες που θέλω και μετά να το χρηsιμοποιήσω με το script του gnuplot και για να το βελτιώσω ώστε να επιλέγω εγώ στήλες και όνομα διαγράμματος τα συνδύασα σε ένα κοινό shell script.

[![asciicast](https://asciinema.org/a/MQLxuWC4iuAU69uRqUi2EANOk.svg)](https://asciinema.org/a/MQLxuWC4iuAU69uRqUi2EANOk)

Το script που δέχεται ως όρισμα τις στήλες για να δημιουργήσει το data.csv
```bash
#!/bin/bash

if [ "$#" -ne 2 ]; then
    echo "Usage: $0 column1 column2"
    exit 1
fi

input_file="mm1-sim.csv"
output_file="data.csv"

column1="$1"
column2="$2"

awk -F, -v col1="$column1" -v col2="$column2" 'BEGIN {OFS=","} {print $col1, $col2}' "$input_file" > "$output_file"

echo "CSV file '$output_file' created with columns $column1 and $column2 from '$input_file'."
```

To κομμάτι αυτό ελέγχει εάν το script έχει παρασχεθεί ακριβώς με δύο ορίσματα (column1 και column2). Εάν ο αριθμός των ορισμάτων είναι διαφορετικός, εμφανίζει το μήνυμα χρήσης και εξέρχεται από το script με έναν κωδικό σφάλματος. Αυτό βοηθά να διασφαλιστεί ότι το script χρησιμοποιείται σωστά με τον αναμενόμενο αριθμό ορισμάτων.


Εδώ δημιουργούνται οι μεταβλητές με το όνομα input_file και output_file

```
column1="$1"
column2="$2"
```
Αυτές οι γραμμές αντιστοιχίζουν το πρώτο και το δεύτερο ορίσματα γραμμής εντολών που ορίστηκαν ($1 και $2, αντίστοιχα) σε μεταβλητές που ονομάζονται στήλη1 και στήλη2.

```
awk -F, -v col1="$column1" -v col2="$column2" 'BEGIN {OFS=","} {print $col1, $col2}' "$input_file" > "$output_file"
```
* awk: Η εντολή για την κλήση του AWK για επεξεργασία κειμένου.
* -F,: Ορίζει το διαχωριστικό πεδίου σε κόμμα, υποδεικνύοντας ότι λειτουργεί με ένα αρχείο CSV όπου οι στήλες διαχωρίζονται με κόμματα.
* -v col1="$column1" -v col2="$column2": Ορίζει δύο μεταβλητές AWK col1 και col2, εκχωρώντας τις τιμές των μεταβλητών column1 και column2 σε αυτές, αντίστοιχα.
* 'BEGIN {OFS=","}: Στο μπλοκ BEGIN, ορίζει το διαχωριστικό πεδίου εξόδου (OFS) σε κόμμα, προσδιορίζοντας ότι η έξοδος θα πρέπει επίσης να είναι CSV.
* '{print $col1, $col2}': Για κάθε γραμμή στο αρχείο εισόδου, εκτυπώνει τις τιμές των στηλών που καθορίζονται από τις μεταβλητές AWK col1 και col2, διαχωρισμένες με κόμμα.
* "$input_file" > "$output_file": Λαμβάνει την είσοδο από το αρχείο που καθορίζεται από τη μεταβλητή input_file και ανακατευθύνει την έξοδο στο αρχείο που καθορίζεται από τη μεταβλητή output_file.
Αυτή η εντολή θα επεξεργαστεί το αρχείο CSV (input_file) με βάση τις στήλες που καθορίζονται από τη στήλη1 και τη στήλη2, δημιουργώντας ένα νέο αρχείο CSV (output_file) με τις επιλεγμένες στήλες.

```bash
#!/bin/bash

input_file="mobydick.html"

  | tr -d '[:punct:][:digit:]' \
```

```lua
function Image(paragraph)
	local to_string = pandoc.utils.stringify
	if paragraph.classes:find("labs", 1) then
		local file = io.open("all_collections/_groups/" .. paragraph.src, "r")
		local content = file:read("*a")
		file:close()

		local doc = pandoc.read(content)

		-- Metadata
		local meta = doc.meta
		local link = to_string(meta.link)
		local title = to_string(meta.title)

		-- process content after metadata
		local body = doc.blocks

		local md = "#### " .. title .. "\n\n" .. link .. "\n\n"

		-- process content blocks
		for _, block in ipairs(body) do
			if block.t == "Para" then
				md = md .. to_string(block)
			elseif block.t == "BulletList" then
				for _, item in ipairs(block.content) do
					md = md .. "- " .. to_string(item) .. "\n\n"
				end
			end
		end
		return pandoc.RawInline("markdown", md)
	end
end
```
